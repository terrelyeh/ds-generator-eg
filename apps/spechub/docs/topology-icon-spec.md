# 產品 Icon 生成規範(拓樸圖用)

> 給設計團隊參考。這些 icon 用於 **Ask SpecHub 自動生成的網路拓樸圖**(application / network topology)。
> Last updated: 2026-06-09

## 核心規則(7 條)

1. **格式**:PNG,**透明背景**(必須)。
2. **尺寸**:**長邊 ≥ 512px**(理想 768–1024px)。判斷看**長邊**,不是寬。不必超過 ~1500px(徒增檔案)。
3. **兩個角度**:
   - `-a` = **正面**
   - `-b` = **側視 iso** ← **拓樸圖用這個,品質最重要**
   - `-c` = 選配(額外角度,拓樸不會用到)
4. **側視 iso 角度全產品統一**:同一個 3/4 視角、同一光源方向。→ 拓樸圖的連接線才會**彼此平行**、整齊。
5. **命名**:`{model}-{view}.png`
   - model 用官方型號、大小寫一致
   - 變體後綴保留:`ECW201L-AC`、`ECP106-INT`、`EOC655-C18`
6. **四周留透明 padding ~5–10%**,避免圖貼邊。
7. **同類設備視覺比例一致**:例如所有 AP 看起來差不多大、線條粗細一致 → 拓樸裡才不會有的大有的小。

## 視角定義

| 後綴 | 視角 | 用途 |
|---|---|---|
| `-a` | 正面 | 其他用途(產品展示等) |
| `-b` | **側視 iso** | **拓樸圖節點(統一角度)** |
| `-c` | 額外角度 | 選配 |

## 命名範例

```
ECW220-a.png      正面
ECW220-b.png      側視 iso（拓樸用）
ECP106-INT-b.png  變體後綴 -INT 保留
EOC655-C18-b.png  變體後綴 -C18 保留
```

## 通用節點(非產品)

拓樸還需要非 EnGenius 的節點,同規則生成,放到對應 Drive 子資料夾:

| 放哪個資料夾 | 內容 | 命名範例 |
|---|---|---|
| `Cloud & Internet/` | Internet 雲、數據機/modem、ISP | `internet-b.png`、`modem-b.png` |
| `General/` | Client / 筆電、手機、Server | `client-b.png`、`server-b.png` |

## 提交方式

把 icon 放進共用 Drive 資料夾(系統會自動同步進 Ask SpecHub):
- **Drive 資料夾**:Product Icons(已分享給 service account `datasheet-sync-723@…`)
- 產品 icon 放 root 即可,**不用分類**(系統依型號前綴自動歸類);通用節點放上表的子資料夾。
