產品 Icon 生成規範

拓樸圖用(Topology Diagram Icons) — 給設計團隊 · 2026-06-09

這些 icon 用於 Ask SpecHub 自動生成的網路拓樸圖。重點:透明 PNG、長邊 ≥ 512px、側視 iso 角度全產品統一(連線才會平行)。

核心規則

  1. 格式:PNG,透明背景(必須)。
  2. 尺寸:長邊 ≥ 512px(理想 768–1024px)。判斷看長邊,不是寬。不必超過 ~1500px。
  3. 兩個角度:-a 正面、-b 側視 iso(拓樸用 -b,最重要)、-c 選配。
  4. 側視 iso 角度全產品統一:同一個 3/4 視角、同一光源方向 → 拓樸連接線才會彼此平行。
  5. 命名:{model}-{view}.png,型號用官方名、大小寫一致;變體後綴保留(-AC / -INT / -C18)。
  6. 四周留透明 padding ~5–10%,避免貼邊。
  7. 同類設備視覺比例一致(所有 AP 差不多大、線條粗細一致)。

視角定義

後綴視角用途
-a正面產品展示等其他用途
-b側視 iso拓樸圖節點(統一角度)
-c額外角度選配,拓樸不用

命名範例

ECW220-a.png      正面
ECW220-b.png      側視 iso（拓樸用）
ECP106-INT-b.png  變體後綴 -INT 保留
EOC655-C18-b.png  變體後綴 -C18 保留

通用節點(非產品)

拓樸還需要非 EnGenius 的節點,同規則生成,放到對應 Drive 子資料夾:

資料夾內容命名範例
Cloud & Internet/Internet 雲、數據機/modem、ISPinternet-b.png
General/Client / 筆電、手機、Serverclient-b.png

提交方式

把 icon 放進共用 Drive 資料夾「Product Icons」(已分享給 service account datasheet-sync-723@…),系統會同步進 Ask SpecHub。

產品 icon 放 root 即可,不用分類 —— 系統依型號前綴自動歸類(ECW→AP、ECS→Switch、ESG→Gateway、ECC→Camera、EVS→NVS、EXT→Extender、ECP/EPC→PDU)。只有通用節點要放上表的子資料夾。
