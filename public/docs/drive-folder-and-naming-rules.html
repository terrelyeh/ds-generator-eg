3Datasheet PDF 命名規則

3.1 子資料夾命名

每個型號在 DS 資料夾內有一個子資料夾：

{DS_Prefix}_{Model}/

範例:

產品線型號資料夾名稱
Cloud APECW526DS_Cloud_ECW526/
Cloud SwitchECS5512FPDS_Cloud_ECS5512FP/
Cloud CameraECC500DS_Cloud_ECC500/
Cloud VPN FWESG510DS_Cloud_ESG510/
Cloud NVSEVS1004DDS_Cloud_EVS1004D/
ExtenderEXT1106DS_Cloud_EXT1106/
Unmanaged SWES105DS_Unmanaged_ES105/
歷史格式: 部分舊資料夾名稱包含版本號（如 DS_Cloud_ECW526_v1.4/），系統可自動辨識。新建的資料夾一律不帶版本號。

3.2 PDF 檔案命名

{DS_Prefix}_{Model}_v{Major}.{Minor}.pdf

範例:

  • DS_Cloud_ECW526_v1.4.pdf
  • DS_Cloud_ECS5512FP_v1.5.pdf
  • DS_Unmanaged_ES105_v1.0.pdf

3.3 版本規則

規則說明
格式v{Major}.{Minor}  例: v1.4
新產品從 v1.0 開始
更新Minor +1  例: v1.4 → v1.5
版本來源系統自動掃描 DS 資料夾中的 PDF 檔名，取最新版本

版本偵測邏輯:

  1. 掃描型號對應的子資料夾
  2. 檢查資料夾名稱中的版本號（舊格式相容）
  3. 檢查資料夾內所有 PDF 檔名中的版本號
  4. 取最大版本號作為 base version
  5. 生成新 PDF 時自動 +1

5Google Sheets 規格表結構

每個產品線有一個 Google Sheet，包含以下頁籤：

頁籤用途資料流向
Web Overview產品名稱、Headline、Overview、Features、Status→ products 表
Detail Specs技術規格 (各 category)→ spec_sections + spec_items
Comparison產品比較表→ comparisons
Revision Log變更紀錄→ revision_logs

5.1 Web Overview 必要欄位

Column A 標籤說明範例
Model Name產品全名Cloud Managed AI Wi-Fi 6 2x2 Indoor Access Point
Model Number型號 (系統 Key)ECW526
Status產品狀態Active / Upcoming / Pending
Headline一行標語High-Performance Tri-Band Wi-Fi 6E Access Point
Single Overview單一段落 Overview(優先使用此欄位)
Key Feature Lists條列式功能 (換行分隔)- Feature 1
- Feature 2

5.2 產品狀態 (Status)

Google Sheet 填寫系統狀態Dashboard 顯示說明
留空 或 Active active 正常顯示 (無標籤) 在售產品，列入 Datasheet 生成
Upcoming upcoming Upcoming 即將上市，規格已建立但尚未發布
Pending pending Pending 暫不發布 (停產/問題/評估中)
篩選: Dashboard 預設只顯示 Active 產品，點右上角「Active / All」按鈕可切換顯示全部。

7自動同步機制

同步排程

方式觸發時間說明
自動每天 09:00 (台灣時間)Vercel Cron 自動觸發，僅同步有變更的產品線
手動隨時Dashboard 右上角「Sync」按鈕，強制同步所有產品線

同步流程

  1. 讀取 Google Sheet (Web Overview + Detail Specs) → 更新產品資料 + 規格
  2. 掃描 DS Images 資料夾 → 同步產品圖片到 Supabase Storage
  3. 掃描 DS 資料夾 → 偵測最新 Datasheet 版本號
  4. 比對現有資料 → 偵測變更 → 記錄 Change Log
  5. 有變更時 → 發送 Telegram 通知

8快速檢查清單

新增一個產品時，需要確認以下事項：

  • Google Sheet Web Overview 填寫完整 (Model Name, Model Number, Status, Headline, Single Overview, Key Feature Lists)
  • Google Sheet Detail Specs 填寫完整
  • DS Images 資料夾放入 {Model}_product.png
  • DS Images 資料夾放入 {Model}_hardware.png
  • (AP only) DS Images 放入 Radio Pattern 圖片
  • 確認 Status 欄位正確 (Active / Upcoming / Pending)
  • 執行一次 Sync 確認資料正確匯入

系統支援為個別產品生成多國語言版本的 Datasheet。每個語言版本有獨立的翻譯內容、版本號、Drive 資料夾。

9.1 支援的語言

語言Locale CodeDrive 資料夾後綴PDF 檔名後綴
🇺🇸 Englishen(無，預設)(無)
🇯🇵 Japaneseja_ja_ja
🇹🇼 Traditional Chinesezh-TW_zh_zh
注意: 繁體中文在系統內使用 zh-TW 作為 locale code，但在 Drive 資料夾和 PDF 檔名中使用簡化的 zh。

9.2 Drive 資料夾結構

每個語言版本有獨立的子資料夾，命名規則為 {DS_Prefix}_{Model}_{locale}/：

9.4 版本獨立管理

每個語言的版本號各自獨立，互不影響：

型號🇺🇸 EN🇯🇵 JA🇹🇼 ZH
ECC100v1.1v1.0—
ECC500v1.0—v1.0
  • Regenerate — 覆蓋該語言的當前版本 PDF
  • New Version — 該語言的 minor 版本 +1（例 v1.0 → v1.1）
  • 版本偵測：系統自動掃描語言資料夾中的 PDF 檔名，取最新版本

9.5 翻譯範圍

翻譯分為兩種模式：

模式翻譯內容不翻譯
Light
  • 頁面標題 (概要、特長・メリット、技術仕様 等)
  • Product Headline (產品大標題)
  • Overview 內文
  • Features & Benefits 條列
  • QR Code 標籤 & 連結
  • Footer 免責聲明
 規格表的 Section Header 和 Spec Label
Full
  • 所有 Light 模式的內容
  • 規格表 Section Header (如 General → 一般仕様)
  • 規格表 Spec Label (如 Operating Temperature → 動作温度)
 Spec Value（數值永遠維持英文）

9.6 Hardware 圖片

Hardware Overview 頁面的圖片可以依語言不同而使用不同版本（如標註文字為日文）。

  • 在翻譯頁面上傳語言專屬的 Hardware Image
  • 有上傳 → 使用該語言版本的圖片
  • 未上傳 → fallback 使用英文版的 Hardware Image

9.7 翻譯工作流程

  1. 進入產品的 Translations tab
  2. 點 + Enable Language 啟用語言（如 Japanese）→ 狀態為 Draft
  3. 翻譯 Product Headline、Overview、Features（手動輸入或 AI 翻譯）
  4. (選) 上傳語言專屬的 Hardware Image
  5. (選) 設定 QR Code 標籤和連結
  6. 點 Preview 確認排版效果
  7. 點 Save & Confirm → 狀態變為 Confirmed
  8. 從 🌐 按鈕 Generate PDF
重要: 必須 Save & Confirm 之後才能 Generate PDF。Preview 可以隨時使用但不算正式確認。

9.8 Spec Label 翻譯（Full 模式）

Spec Label 翻譯是 per-product-line（產品線層級），同一產品線的所有型號共用同一套翻譯。

  • 入口：Dashboard toolbar → Translations 連結
  • 每個產品線獨立一個管理頁面
  • 包含 Section Header 翻譯 + Spec Label 翻譯
  • 修改會影響該產品線所有型號的 Full 模式 Datasheet

9.9 AI 翻譯 & 詞庫

系統內建 AI 翻譯功能，支援多家 AI 模型：

Provider模型需要 API Key
AnthropicClaude Sonnet 4.6 / Opus 4.6ANTHROPIC_API_KEY
OpenAIGPT-4oOPENAI_API_KEY
Google AIGemini 2.5 ProGOOGLE_AI_API_KEY

API Key 可以在 Settings 頁面設定（存到 DB），或透過 Vercel 環境變數設定。Settings 優先於 env var。

AI 翻譯特點:

  • 翻譯同時改善原文品質（修正不通順的句子結構、統一術語）
  • 5 層 Prompt 系統：基礎翻譯規則 → 語言規則 → 產品線術語 → 內容類型規則 → 公司詞庫
  • 翻譯完成後顯示翻譯筆記（用繁體中文說明做了哪些優化）
  • AI Model 下拉只顯示已設定 API Key 的模型，未設定的會反灰

翻譯詞庫 (Glossary):

  • 在 Settings → Manage Glossary 管理公司認可的翻譯術語
  • 詞庫分 Global（全公司通用）和產品線專屬（如只套用於 Cloud Camera）
  • AI 翻譯時自動載入詞庫，確保術語一致性
  • 範例：Night Vision → ナイトビジョン (JA) / 夜視 (ZH)

9.10 CJK 排版

日文 / 中文版 Datasheet 自動套用以下排版規則：

  • 禁則處理 (line-break: strict) — 句號、逗號不出現在行首
  • 兩端對齊 (text-align: justify) — 右邊不參差不齊
  • CJK 字型 — 日文使用 Noto Sans JP，繁中使用 Noto Sans TC
  • 行高和字級針對 CJK 文字微調，確保可讀性
